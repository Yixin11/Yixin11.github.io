对象的生命周期

联想一下你身边的电子设备，手机、PC、iPad 以及耳机，你的机器和别人的同一款在硬件上并没有什么不同。每一类产品都是由同一套模具和制作工艺批量化生产的，所以无论你购买哪一台产品都无所谓，只要你选择正确你所需要的型号就可以。

这些产品大多设计为三个生命周期：被工厂生产（创建）、被用户使用（使用）、被工厂回收（销毁）。

现在让我们将注意力放回 C++。在面向对象的 C++ 中，对象的生命周期也同样被分为“创建”、“使用”和“销毁”。学习如何正确且安全的创建和销毁对象是用好 C++ 的第一步，至于使用部分，则会分散到所有的地方去讲。

正确且安全的创建对象

试图访问一个没有初始化的对象是一件非常危险的事情，因为你永远不知道会发生什么事，这是一种不确定行为。在我们正确的使用任何变量前，必须要将对象正确的初始化。

首先我们应该知道 C++ 初始化一个变量或对象的方法。初始化一个变量可以使用 operator= 进行赋值，也可以使用 typename variable(value) 在创建时直接创建一个变量。

初始化一个对象需要分为三种情况：

  • 调用构造函数

  • 调用拷贝构造函数

  • 赋值
