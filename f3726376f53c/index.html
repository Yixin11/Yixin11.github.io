Linux 介绍及使用

Posted on 2020-11-22 In Linux
简单整理一下 Linux 开发相关，偏向介绍、简单学习和使用。

有时候面试的时候会被问到会不会用 Linux，有时思路比较混乱，所以整理一下。

Linux 基础

Linux 是 Linus 大神开源的一套类 Unix 操作系统，之后经过各路大神的齐心协力、夜以继日的开发，目前取代了 Unix 成为了商业上广泛运用的 OS。这位大神还花了一周开发了 Git。

将 Linux 拆开来看就是 Linux 内核 + 发行版 + shell + 图形化界面。

内核版本是统一的，在此基础上会推出不同的发行版，比如 Centos、 Ubuntu、 arch-linux、Manjaro 和 deepin 等（deepin 真的难用）。

shell 也可以用很多，脚本一般使用 #!/bin/sh 保证执行环境的统一，私人机器上会用 zsh。

图形化界面主要是 xfce、Gnome 和 KDE，我用的是 KDE，美化比较好看。

但是平时不将 Linux 作为主力机，主力机是 Windows (for Steam)。

在电脑上安装 Linux 发行版

现在可以很方便的制作 Linus 启动盘，第一件事其实是选择你想要的操作系统，新人建议学习可以用 Centos，也可以用 Ubuntu 或者 Manjaro；进阶的话可以装 arch-linux 用。去开源镜像站可以下载相关镜像，常见的有腾讯云、阿里云、华为云、清华大学镜像站、浙江大学镜像站和上海交大镜像站。然后使用 refus 这款软件镜像刻录。我之前用软碟通，破解还要额外百度一次。

启动之后可以直接从 U 盘启动，然后装到磁盘里。现在还会很体贴的提供自动分区功能，不装什么数据划 50G 左右就够了。

需要注意的是上面提到的 Centos、Ubuntu 和 Manjaro 等都是 Linux 的发行版，是基于 Linux 内核开发的。而 Linux 是只有命令行界面的，如果需要图形化界面需要额外配置图形软件，常用的是 Gnome 和 KDE 桌面。

当然面试不会聊这些东西，运维都是直接申请一台 openstack 虚拟机，账号密码一甩就完事了。但是计算机专业一定要会自己重装系统，不管是面对七大姑八大姨各种崩溃问题，还是因为打游戏要重装，这是表现自己的好时机。一般在异性同学面前我会表示不会，一是对方以后电脑出问题都会找你，二是电脑修的好备胎当到老。我对面床那哥们帮很多女生重装系统了，现在和我一样还是条单身狗，呵。

Linux 指令

有的时候面试，对方问：你都知道什么 Linux 指令…其实感觉是有点尴尬的，因为 ls 应该也算个指令…

决定以后这么回答：

  • CPU 方面：使用 top 可以看到，按 shift -f 可以切换多种排序
  • 内存方面：free
  • 网络方面：netstat -anp（查看进程所有端口号），lsof -i | grep
  • 进程方面：ps 查看进程端口号，pstree 查看进程关系
  • 查找指令：awk 替换变量， sed 替换行， grep 字符匹配
  • 其他指令：wc -l 显示行数，很实用；nc 指令，又称瑞士军刀，很好用； vmstat 看一下虚拟机状态；定时脚本 crontab；
  • 各种自定义命令

最主要的还是为什么要用到这些指令。因为有很多信息存在这些指令里面。

比如知道进程号再去主动 kill -7 或者 -9 进程；比如有时候要知道 CPU 和内存的负载情况，确保系统的良好运行；比如有时候要传输文件，抓包 wrk tcp port，或者屏蔽无效信息。

所以，Linux 指令是如何执行的？

Linux Shell

Shell 直译是壳，它也是 Linux 外壳。

操作系统是硬件和用户的中间层，主要提供向下管理硬件资源和向上兼容软件和用户的功能。Linux 内核会自动管理 CPU、内存、磁盘和文件资源，Shell 则提供用户交互功能。

Shell 主要写脚本，辅助 crontab 进行定时任务。

语法有点独特，毕竟是早期计算机语言；但是结构简单，好理解，也很好用。这方面目前只能写一些简单的脚本，需要继续学习。

Linux 开发环境配置

有的时候确实会觉得 Linux 比较麻烦，比如没有 Linux for QQ 这种东西，要先装个 wine 才能用微信和 QQ。虽然有 Steam 可以买买买，但是很多游戏不支持 Linux 游玩，这样就真的只能成为喜加一平台。

如果可以在 windows 环境下写代码，然后把代码拷贝到 linux 环境下编译运行就好了。

这样做比较麻烦，因为每次还要打包上传解压，最主要的是任何修改都要这样做一遍。如果主动修改 Linux 端的代码，就会涉及到代码不同步的现象。

Jerbrains 在 Pycharm 提供了内置的 Linux 编译环境，算是一种解决方案。但是鹅的正版 Jetbrains 联网激活 key 完!全!不!够!用!!!

所以使用的是 vscode + remote 的方式，直连开发环境，直接调试。

后来被 leader 抓了用 datagrip 直连数据库，说不让直连 db 防止泄露，因为用的是盗版 datagrip（鹅不买 datagrip）。

但是 JetBrains 确实很好用很香。
